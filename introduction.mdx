## What Rose is

Rose monitors the League Client via the League Client Update (LCU) API. When you hover a skin in champion select, a JavaScript plugin called ROSE-SkinMonitor sends the selection to Rose's Python backend over a WebSocket connection. Rose then decrypts the skin file at runtime, injects it when the game process starts, and wipes the decrypted data after use. The skin loads as if you owned it, with no impact on game behaviour or other players.

Rose is built on the [Pengu Loader](https://github.com/Tariolle/ROSE-Pengu) framework, which injects a suite of JavaScript plugins into the League Client. These plugins unlock locked skin previews, display chroma and form selectors, surface a settings panel, and enable real-time skin sharing between friends in party mode.

## Key features

- **Smart injection** — Rose never injects skins you already own; it only applies what you're missing
- **Chroma and forms support** — Choose any chroma variant or alternate form directly in champion select
- **Party mode** — Share skin selections with friends in the same lobby in real time via a Cloudflare WebSocket relay
- **Client customization** — Customize regalia borders, banners, icons, titles, and win/loss stats via the ROSE-Jade plugin
- **Historic and random skins** — Quickly re-apply your last-used skin or pick a random one per champion
- **Auto-updater** — On startup, Rose checks GitHub for new releases and installs updates automatically
- **Multi-language support** — Works with any League Client language

## Architecture overview

Rose is composed of three layers that work together:

**Python backend** — The core application runs as a system tray process. It connects to the LCU API, manages a WebSocket server for plugin communication, handles skin downloads and decryption, runs the auto-updater, coordinates game monitoring, and orchestrates the injection pipeline.

**Pengu Loader plugins** — A set of JavaScript plugins injected into the League Client handle all UI interactions: unlocking skin previews (ROSE-UI), monitoring selections (ROSE-SkinMonitor), providing chroma and form pickers (ROSE-ChromaWheel, ROSE-FormsWheel), exposing settings (ROSE-SettingsPanel), and enabling party mode (ROSE-PartyMode).

**Cloudflare Workers** — Two workers support the backend: `rose-party-relay` is a Durable Object-backed WebSocket relay that manages party rooms for real-time skin sharing, and `rose-skin-key` serves the decryption key for skin files at runtime.
